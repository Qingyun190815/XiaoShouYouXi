# 文字探险游戏技能创建规范

一、技能概述

本文档定义了创建「文字探险游戏生成器」技能的技术规范和完整流程。该技能用于自动生成分支叙事型文字冒险游戏。

---

二、代码约定规范

2.1 技能 ID 与命名规范

| 规范项 | 要求 |
| --- | --- |
| 技能 ID 格式 | text_adventure_{难度}_{序号}，如 text_adventure_medium_001 |
| 技能名称 | 清晰描述功能，如「文字探险游戏生成器」 |
| 版本号 | 使用语义化版本，如 v1.0.0 |
| 语言 | 技能内部说明统一使用中文 |

2.3 文本规范

| 规范项 | 要求 |
| --- | --- |
| 场景描述 | 50-200 字，生动具体有画面感 |
| 选项文本 | 简洁明确，10-30 字 |
| 后果描述 | 说明选择造成的影响 |
| 结局文本 | 100-300 字，含结局感言 |
| 标点符号 | 使用中文标点（，。！？） |
| 行文风格 | 第三人称描写，文学性强 |

2.5 错误处理规范

| 错误码 | 说明 | 处理方式 |
| --- | --- | --- |
| E001 | 主题为空 | 返回错误提示，要求补充主题 |
| E002 | 题材不支持 | 返回支持题材列表 |
| E003 | 场景数超限 | 自动压缩或提示用户调整 |
| E004 | 分支过深 | 限制分支层数 |
| E005 | 生成超时 | 返回部分内容并提示 |

2.6 操作历史与回退规范

功能要求

| 规范项 | 要求 |
| --- | --- |
| 历史记录 | 每次玩家做出选择时，记录操作前的完整状态快照 |
| 撤销操作 | 支持撤销最近一次选择，恢复到操作前的状态 |
| 指定回退 | 支持跳转到任意历史记录点，回退到该操作前的状态 |
| 回退策略 | 采用分支覆写模式——回退后，被回退部分的历史记录将被丢弃 |
| 持久化 | 操作历史随游戏状态一起保存至 localStorage |
| 向后兼容 | 加载不含操作历史的旧存档时，操作历史默认为空数组 |

动作类型规范

| 动作类型 | 参数 | 说明 |
| --- | --- | --- |
| UNDO_CHOICE | 无 | 撤销最近一次选择，恢复快照状态 |
| ROLLBACK_TO | targetIndex: number | 回退到指定索引的历史记录点 |

界面规范

| 元素 | 位置 | 说明 |
| --- | --- | --- |
| 撤销按钮 | 页面头部，游戏标题旁 | 点击撤销最近选择，无可撤销时禁用 |
| 操作历史面板 | 侧栏，道具/线索/关系上方 | 显示所有历史选择，点击可回退到该点 |
| 键盘快捷键 | Ctrl+Z / Cmd+Z | 快速撤销最近选择 |
| 结局页撤销 | 结局画面底部 | 从结局回退到最后一个选择点 |

2.7 成就系统规范

功能要求

| 规范项 | 要求 |
| --- | --- |
| 成就定义 | 在游戏数据中定义成就列表，包含条件类型和参数 |
| 条件检查 | 每次玩家做出选择后，自动检查是否达成新成就 |
| 来源记录 | 记录成就达成时的场景、选项和来源描述 |
| 隐藏成就 | 支持隐藏成就，未解锁时不显示标题（显示"???"） |
| 跨存档累计 | all_endings 类型成就跨存档累计，存储在独立 localStorage 键 |
| 通知提示 | 成就解锁时在页面右上角显示滑入通知，4秒后自动消失 |
| 持久化 | 已完成成就随游戏状态保存至 localStorage |
| 向后兼容 | 加载不含成就的旧存档时，已完成成就默认为空数组 |

动作类型规范

| 动作类型 | 参数 | 说明 |
| --- | --- | --- |
| UNLOCK_ACHIEVEMENT | achievement: CompletedAchievement | 解锁成就，已存在则跳过 |

界面规范

| 元素 | 位置 | 说明 |
| --- | --- | --- |
| 成就面板 | 侧栏，操作历史下方 | 显示所有可见成就，已解锁高亮，点击展开来源详情 |
| 解锁通知 | 页面右上角浮层 | 成就解锁时滑入显示，4秒自动消失 |
| 结局页成就 | 结局画面中 | 显示本次游戏达成的成就列表 |
| 头部计数 | 页面头部状态栏 | 显示已达成/总成就数 |

三、创造流程规范

3.1 流程总览

`┌─────────────────┐ │ Step 1: 接收需求 │ └────────┬────────┘ ▼ ┌─────────────────┐ │ Step 2: 需求分析 │ └────────┬────────┘ ▼ ┌─────────────────┐ │ Step 3: 世界观构建 │ └────────┬────────┘ ▼ ┌─────────────────┐ │ Step 4: 角色设计 │ └────────┬────────┘ ▼ ┌─────────────────┐ │ Step 5: 场景规划 │ └────────┬────────┘ ▼ ┌─────────────────┐ │ Step 6: 剧情编写 │ └────────┬────────┘ ▼ ┌─────────────────┐ │ Step 7: 结局设计 │ └────────┬────────┘ ▼ ┌─────────────────┐ │ Step 8: 状态系统 │ └────────┬────────┘ ▼ ┌─────────────────┐ │ Step 9: 格式输出 │ └────────┬────────┘ ▼ ┌─────────────────┐ │ Step 10: 质量校验 │ └────────┬────────┘ ▼ ┌─────────────────┐ │ 完成交付 │ └─────────────────┘`

3.2 详细流程

Step 1: 接收需求

输入：用户提供的基本信息

| 用户输入 | 说明 | 必填 |
| --- | --- | --- |
| 游戏主题 | 想做什么样的游戏 | 是 |
| 题材偏好 | 奇幻/科幻/悬疑/恐怖/历史/现代 | 是 |
| 时长要求 | 短篇/中篇/长篇 | 是 |
| 主角设定 | 主角身份和背景（可选） | 否 |
| 特殊需求 | 希望包含的元素（可选） | 否 |

输出：标准化的需求对象

---

Step 2: 需求分析

任务：

1. 解析用户输入，提取关键信息

2. 确定游戏规模参数

3. 检查需求完整性

4. 如有缺失，提示用户补充

规模参数对照：

| 长度 | 场景数 | 分支数 | 结局数 | 预计字数 |
| --- | --- | --- | --- | --- |
| 短篇 | 3-5 | 2-3 | 2-3 | 2000-5000 |
| 中篇 | 10-20 | 3-4 | 3-7 | 5000-15000 |
| 长篇 | 20+ | 4+ | 5-10 | 15000+ |

输出：需求分析报告

---

Step 3: 世界观构建

任务：

1. 确定时代背景和地域设定

2. 设计主要势力和阵营（2-4 个）

3. 设定核心冲突和主题

4. 创建独特的 世界观元素

输出：

时代背景\

- 势力 B：【描述】

【描述】\
`

Step 4: 角色设计

任务：

1. 设计主角身份和背景

2. 创建主要 NPC（3-6 个）

3. 设定各角色动机和个性

4. 规划角色在故事中的作用

5. 设定角色间关系

输出：

`markdown \

- NPC1：【角色描述】

  `

Step 5: 场景规划

任务：

1. 设计场景网络拓扑

2. 确定场景间的连接关系

3. 规划关键场景和转折点

4. 分配场景到故事节点

场景类型：

| 类型 | 说明 | 示例 |
| --- | --- | --- |
| 起始场景 | 游戏开头 | 玩家进入世界 |
| 探索场景 | 可自由探索 | 城镇、野外 |
| 事件场景 | 触发剧情 | 遭遇战、对话 |
| 关键场景 | 重要转折 | BOSS 战、选择 |
| 结局场景 | 通往结局 | 最终决策 |

输出：场景网络图

`[start] ──> [scene_01] ──> [scene_02] ──> [ending_1] │ └─> [scene_03] ──> [scene_04] ──> [ending_2] │ └─> [ending_3]`

Step 6: 剧情编写

任务：

1. 编写每个场景的描述文本（50-200 字）

2. 设计 2-4 个分支选项

3. 编写每个选项的后果描述

4. 关联状态变化

编写规范：

`markdown \

【场景描述，50-200 字】

- 符合角色设定

### 6.2 核心生成器

```typescript
class TextAdventureGenerator {
  async generate(params: GenerationParams): Promise<game> {
    // Step 1: 世界观构建
    const worldSetting = await this.buildWorldSetting(params);
    
    // Step 2: 角色设计
    const characters = await this.designCharacters(params);
```
